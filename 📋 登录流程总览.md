

## 📋 登录流程总览

### 一、页面加载阶段（自动认证检查）

**触发点**：页面加载完成后自动调用 `checkAuthConfig()` (第941行)

```
页面加载 → checkAuthConfig() → /api/auth/config (GET)
```

**处理逻辑** (`checkAuthConfig` 函数，第849-880行)：

| 返回结果                                                     | 处理方式                                           |
| ------------------------------------------------------------ | -------------------------------------------------- |
| `code=200` + `autoAuthSuccess=true`                          | ✅ 环境变量凭据自动认证成功，直接进入系统           |
| `code=200` + `autoAuthSuccess=false` + `hasCredentials=true` | ⚠️ 环境变量配置的凭据认证失败，预填用户名并提示错误 |
| `code=200` + `autoAuthSuccess=false` + `hasCredentials=false` | 📝 显示空白登录框，等待用户手动输入                 |
| 请求失败/异常                                                | 📝 显示登录框                                       |

### 二、手动登录流程

**触发点**：用户点击登录按钮，调用 `login()` (第688行)

```
用户输入 → login() → 验证输入 → /api/auth (POST) → 处理结果
```

**详细步骤** (`login` 函数，第883-918行)：

1. **输入验证**：检查用户名和密码是否为空
   ```javascript
   if (!username || !password) {
     errorEl.textContent = '请输入用户名和密码';
     return;
   }
   ```

2. **发起登录请求**：
   ```javascript
   fetch('/api/auth', {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({ username, password })
   });
   ```

3. **处理响应**：
   - ✅ **成功** (`code === 200`)：
     - 设置 `isAuthenticated = true`
     - 隐藏登录页面 (`login-page`)
     - 显示设备总览页面 (`overview-page`)
     - 调用 `initConfig()` 初始化系统配置
   
   - ❌ **失败**：
     - 显示错误提示信息 (`result.msg` 或 "登录失败")

### 三、登录成功后的初始化

**调用链**：`login()` → `initConfig()` (第909行)

**`initConfig` 函数** (第921-937行)：

1. **获取服务端配置**：
   ```javascript
   fetch('/api/config');
   ```

2. **更新全局变量**：
   - `allDeviceConfigs = config.configs` - 设备配置信息
   - `wsUrl` - WebSocket连接地址（如果服务端返回了自定义端口）

3. **连接WebSocket**：
   ```javascript
   connectWebSocket();
   ```
   - 连接到 `ws://hostname:8084`（或服务端配置的端口）
   - 监听设备状态更新消息

## 📌 关键要点

1. **双重认证机制**：支持环境变量自动认证 + 手动用户名密码认证
2. **认证状态标记**：使用全局变量 `isAuthenticated` 记录登录状态
3. **无状态会话**：登录成功后直接切换页面，没有使用Token或Session持久化
4. **WebSocket连接**：登录成功后立即建立WebSocket连接，用于实时接收设备状态更新
5. **错误提示**：登录失败时显示详细错误信息，环境变量凭据失败时预填用户名
