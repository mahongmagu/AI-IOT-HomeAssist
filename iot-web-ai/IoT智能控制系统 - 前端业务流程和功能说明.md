## 二、核心功能模块

### 2.1 设备控制页面（index.html）

#### 2.1.1 设备组总览

**功能描述**：以卡片网格形式展示所有设备组，显示设备组状态、位置信息

**业务流程**：
1. 页面加载时调用 `/api/devices` 获取设备配置
2. 根据设备状态动态显示卡片颜色
3. 支持短按进入详情、长按弹出快捷菜单

**卡片状态颜色规则**：

| 颜色   | 状态含义       | 判定条件                     |
| :----- | :------------- | :--------------------------- |
| 🟢 绿色 | 全部在线且开启 | 所有control类型单元状态为ON  |
| 🟠 橙色 | 部分开启       | 部分control类型单元状态为ON  |
| ⚫ 黑色 | 全部关闭       | 所有control类型单元状态为OFF |
| ⚪ 灰色 | 离线           | 设备无状态数据或离线         |

#### 2.1.2 设备详情

**功能描述**：展示设备组下所有设备单元的详细信息和操作按钮

**设备单元类型**：

| 类型       | 标识      | 控制方式     | 操作按钮  |
| :--------- | :-------- | :----------- | :-------- |
| **控制型** | `control` | ON/OFF开关   | 打开/关闭 |
| **状态型** | `state`   | 文本状态展示 | 查询状态  |
| **文本型** | `text`    | 文本编辑     | 编辑文本  |
| **数据型** | `data`    | 数值编辑     | 编辑数值  |

**业务流程**：
```
点击设备组卡片 → 加载设备单元列表 → 显示各单元状态 → 用户操作 → 发送控制指令
```

#### 2.1.3 实时状态同步

**功能描述**：通过WebSocket实时接收设备状态变化

**技术实现**：
- WebSocket连接地址：`ws://<server-ip>:8084`
- 消息格式：`{ "type": "state-update", "device": {...}, "state": "ON/OFF", "timestamp": ... }`
- 连接状态显示：底部状态栏实时显示连接状态

**状态推送流程**：
```
设备端 → MQTT服务器 → 状态服务 → WebSocket → 前端页面 → UI更新
```

#### 2.1.4 AI指令控制

**功能描述**：支持自然语言指令控制设备

**接口调用**：
- POST `/api/ai` - 基础AI接口
- POST `/api/ai-advanced` - 高级AI接口

**使用方式**：
1. 在AI控制面板输入指令（如"打开插座1"、"关闭风扇"）
2. 点击"发送给AI"按钮
3. 显示AI解析结果和执行状态

#### 2.1.5 定时任务管理

**功能描述**：为控制型设备设置定时任务

**定时类型**：

| 类型   | 说明             | 配置项                   |
| :----- | :--------------- | :----------------------- |
| 一次性 | 指定时间执行一次 | 执行时间、命令           |
| 每天   | 每天固定时间执行 | 执行时间、命令           |
| 每周   | 指定星期执行     | 星期选择、执行时间、命令 |
| 周期性 | 每隔N分钟执行    | 周期(分钟)、命令         |

**业务流程**：
```
长按设备组卡片 → 选择"定时设置" → 选择设备单元 → 配置定时参数 → 保存任务
```

**任务管理操作**：
- 添加定时任务
- 编辑已有任务
- 删除任务
- 启用/禁用任务

### 2.2 配置管理页面（config.html）

#### 2.2.1 设备组管理

**功能描述**：设备组的增删改查操作

**设备组数据结构**：
```json
{
"name": "设备组名称(英文)",
"displayName": "显示名称(中文)",
"id": "自动生成的ID",
"location": "位置(可选)",
"units": []
}
```

**操作流程**：

| 操作       | 接口                         | 说明           |
| :--------- | :--------------------------- | :------------- |
| 查询列表   | GET `/api/devices`           | 获取所有设备组 |
| 创建设备组 | POST `/api/devices`          | 创建新设备组   |
| 查询详情   | GET `/api/devices/{name}`    | 获取指定设备组 |
| 更新设备组 | PUT `/api/devices/{name}`    | 更新设备组信息 |
| 删除设备组 | DELETE `/api/devices/{name}` | 删除设备组     |

#### 2.2.2 设备单元管理

**功能描述**：在设备组下添加、编辑、删除设备单元

**设备单元数据结构**：
```json
{
"id": "自动生成的单元ID",
"name": "设备单元名称",
"type": "control|state|text|data",
"status": "ON|OFF|其他状态值"
}
```

**操作流程**：

| 操作     | 接口                                             | 说明                 |
| :------- | :----------------------------------------------- | :------------------- |
| 添加单元 | POST `/api/devices/{groupName}/units`            | 在指定设备组添加单元 |
| 更新单元 | PUT `/api/devices/{groupName}/units/{unitId}`    | 更新单元信息         |
| 删除单元 | DELETE `/api/devices/{groupName}/units/{unitId}` | 删除单元             |

## 四、数据流说明

### 4.1 数据流向图

```
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│   前端页面    │      │    后端服务   │      │   MQTT服务器 │
│  (index.html)│      │   (Node.js)   │      │   (EMQX)     │
└──────┬───────┘      └──────┬───────┘      └──────┬───────┘
       │                     │                     │
       │ WebSocket(8084)    │                     │
       │←── 状态推送 ────────│                     │
       │                     │                     │
       │ REST API(3002)     │                     │
       │──→ 控制指令 ───────→│                     │
       │                     │ MQTT TCP(1883)     │
       │                     │──→ 控制指令 ───────→│
       │                     │                     │
       │                     │←── 状态上报 ────────│
       │                     │                     │
       │                     │                     │
       │                     │                     │
```

### 4.2 MQTT主题设计

| 主题                              | 方向          | 用途         |
| :-------------------------------- | :------------ | :----------- |
| `iot/device/{unitId}/control`     | 服务端→设备端 | 控制指令下发 |
| `iot/device/{unitId}/state`       | 设备端→服务端 | 状态上报     |
| `iot/device/{unitId}/textsetting` | 双向          | 文本设置     |
| `iot/device/{unitId}/datasetting` | 双向          | 数据设置     |

## 五、交互特性

### 5.1 触摸/鼠标事件支持

| 操作         | 触发方式                   | 效果                       |
| :----------- | :------------------------- | :------------------------- |
| 短按/点击    | 触摸<400ms 或 鼠标点击     | 进入设备组详情页           |
| 长按         | 触摸≥400ms                 | 弹出快捷菜单（定时设置）   |
| 电源按钮点击 | 点击设备组卡片上的电源按钮 | 切换该组所有控制型设备状态 |

### 5.2 响应式设计

- **桌面端**：多列网格布局，支持鼠标操作
- **移动端**：2列网格布局，支持触摸操作
- **底部状态栏**：固定显示连接状态、设备统计

## 六、状态管理

### 6.1 前端状态变量

| 变量                 | 类型      | 说明                            |
| :------------------- | :-------- | :------------------------------ |
| `allDeviceConfigs`   | Object    | 所有设备组配置信息              |
| `allDevices`         | Object    | 所有设备实时状态                |
| `currentView`        | String    | 当前视图（'overview'/'detail'） |
| `currentDeviceGroup` | String    | 当前选中的设备组名称            |
| `ws`                 | WebSocket | WebSocket连接实例               |
| `menuVisible`        | Boolean   | 长按菜单显示状态                |

### 6.2 状态同步机制

1. **初始加载**：页面初始化时获取所有设备配置和初始状态
2. **实时更新**：通过WebSocket接收状态变更推送
3. **本地缓存**：设备状态缓存在内存中，支持快速UI渲染

## 七、错误处理

### 7.1 连接错误处理

| 错误场景          | 处理方式      | 用户提示           |
| :---------------- | :------------ | :----------------- |
| WebSocket连接失败 | 3秒后自动重连 | 状态栏显示"未连接" |
| API请求失败       | 显示错误信息  | 弹窗提示错误原因   |
| MQTT消息发送失败  | 重试机制      | 操作失败提示       |

### 7.2 用户操作反馈

- 操作成功：弹窗提示成功信息
- 操作失败：弹窗提示失败原因
- 加载状态：显示加载指示器

## 八、技术特点

| 特性           | 说明                                |
| :------------- | :---------------------------------- |
| **无框架依赖** | 纯HTML/CSS/JavaScript实现，轻量高效 |

| **MQTT集成**   | 通过后端服务桥接MQTT协议            |
| **响应式布局** | 适配桌面端和移动端                  |
