# IoT Web AI 项目今日变更总结（2026-05-08）

## 一、定时任务功能修复

### 1. 日任务和周任务不执行问题
**问题**：创建的日任务和周任务无法在指定时间执行，而是立即开始执行
**原因**：后端创建周期性任务时没有保存 `time` 字段，直接使用 `setInterval` 立即开始
**修复**：
- 前端创建日/周任务时保存具体执行时间（`data.time`）
- 后端接收并保存 `time` 字段
- 修改定时器逻辑：先计算到下次执行的延迟，等待到指定时间后再创建循环定时器

### 2. 修改任务时 time 字段不更新问题
**问题**：修改日/周任务的执行时间后，保存的时间不变
**原因**：后端更新任务时只更新了 `interval`，没有更新 `time` 字段
**修复**：在更新周期性任务时同时更新 `time` 字段

### 3. 重启服务后任务不显示问题
**问题**：重启服务后，设备单元中不显示定时任务
**原因**：`initSchedules` 函数只恢复了启用的任务到内存，禁用的任务没有加载
**修复**：先将所有任务加载到内存，再对启用的任务恢复定时器

### 4. 修改任务时设备单元下拉框为空问题
**问题**：点击修改任务时，设备单元下拉框没有选项
**原因**：`editSchedule` 函数没有先填充下拉框就设置选中值
**修复**：在设置选中值之前先遍历所有设备配置填充下拉框

### 5. 删除任务时空指针错误
**问题**：删除任务时报错 `Cannot read properties of null (reading 'classList')`
**原因**：代码尝试访问不存在的 DOM 元素 `schedule-list-modal`
**修复**：先检查元素是否存在，再访问其属性

### 6. toggleScheduleEnable 函数错误
**问题**：启用/禁用任务时报错 `showDeviceDetail is not defined`
**原因**：调用了不存在的 `showDeviceDetail` 函数
**修复**：改为调用正确的 `loadDeviceDetail` 函数

### 7. 后端 timer 变量未定义错误
**问题**：创建周期性任务时服务器崩溃（500错误）
**原因**：代码在 `setTimeout` 回调外使用了回调内定义的 `timer` 变量
**修复**：使用 `startTimer` 替代未定义的 `timer`

## 四、数据格式规范

### 定时任务存储格式（`./data/schedules.json`）
```json
{
  "设备单元ID-时间戳": {
    "id": "设备单元ID-时间戳",
    "name": "位置-设备组名称-设备单元名称-序号",
    "unitId": "设备单元ID",
    "cmd": "ON/OFF",
    "interval": 86400000,  // 日任务=86400000，周任务=604800000，周期任务=自定义毫秒
    "time": "2026-05-09T08:00:00.000Z",  // 日/周任务必须有，指定具体执行时间
    "enabled": true,
    "createdAt": "2026-05-08T10:21:59.650Z",
    "recurring": true
  }
}
```

### 字段说明
| 字段        | 说明                                                         |
| ----------- | ------------------------------------------------------------ |
| `interval`  | 执行间隔：日任务=86400000（24小时），周任务=604800000（7天） |
| `time`      | **必须有！** 第一次执行的具体时间（ISO格式）                 |
| `recurring` | true（表示周期性任务）                                       |
| `enabled`   | true/false（控制任务是否启用）                               |
