## 二、数据模型

### 2.1 设备组结构 (Device Group)

```javascript
{
  "name": "shg",                    // 设备组标识名（英文，唯一）
  "displayName": "珊瑚缸",           // 显示名称（中文）
  "id": "keting-shg-64771774",      // 系统自动生成的唯一ID
  "location": "客厅",                // 位置信息
  "units": [...]                    // 设备单元列表
}
```

**ID生成规则**: `{位置拼音}-{设备名}-{8位随机数字}`

### 2.2 设备单元结构 (Device Unit)

```javascript
{
  "id": "keting-shg-64771774_fengshan-0838",  // 单元唯一ID
  "name": "风扇",                              // 单元名称
  "status": "ON",                              // 当前状态
  "type": "control"                            // 单元类型
}
```

**ID生成规则**: `{设备组ID}_{单元名称拼音}-{4位随机数字}`

### 2.3 设备单元类型

| 类型   | 标识    | 说明             | 状态输入方式    |
| ------ | ------- | ---------------- | --------------- |
| 控制型 | control | 可控制的开关设备 | ON/OFF 下拉选择 |
| 状态型 | state   | 只读传感器状态   | 文本输入        |
| 文本型 | text    | 文本数据显示     | 文本输入        |
| 数据型 | data    | 数值配置参数     | 数字输入        |

## 三、功能模块

### 3.1 设备组管理

| 功能           | API接口                   | 方法   | 说明             |
| -------------- | ------------------------- | ------ | ---------------- |
| 获取所有设备组 | `/api/devices`            | GET    | 返回完整设备配置 |
| 获取单个设备组 | `/api/devices/:groupName` | GET    | 根据名称查询     |
| 创建设备组     | `/api/devices`            | POST   | 自动生成ID       |
| 更新设备组     | `/api/devices/:groupName` | PUT    | 保留原始ID       |
| 删除设备组     | `/api/devices/:groupName` | DELETE | 级联删除单元     |

#### 3.1.1 获取所有设备组

| 属性         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **API接口**  | `/api/devices`                                               |
| **请求方法** | GET                                                          |
| **功能说明** | 获取系统中所有设备组的完整配置信息，包括每个设备组的名称、显示名称、ID、位置和所属设备单元列表。用于前端初始化加载设备列表。 |
| **操作说明** | 前端页面加载时自动调用，无需传入参数。服务端读取 `devices.json` 文件，过滤掉 `lastUpdated` 元数据字段后返回设备组对象集合。 |
| **返回示例** | {<br/>  "code": 200,<br/>  "data": {<br/>    "sdj": {<br/>      "name": "sdj",<br/>      "displayName": "扫地机",<br/>      "id": "baomujian-sdj-40858013",<br/>      "location": "保姆间",<br/>      "units": [...]<br/>    },<br/>    "shg": {...}<br/>  },<br/>  "msg": "获取设备配置成功"<br/>} |



#### 3.1.2 获取单个设备组

| 属性         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **API接口**  | `/api/devices/:groupName`                                    |
| **请求方法** | GET                                                          |
| **功能说明** | 根据设备组名称（英文标识）获取单个设备组的详细配置信息。用于查看特定设备组的完整信息，或在编辑前获取原始数据。 |
| **操作说明** | 在URL路径中传入设备组名称（如 `/api/devices/shg`）。服务端检查该名称对应的设备组是否存在，存在则返回详细信息，不存在则返回404错误。 |
| **参数说明** | `groupName` - 设备组英文名称（必需，路径参数）               |
| **返回示例** | {<br/>  "code": 200,<br/>  "data": {<br/>    "name": "shg",<br/>    "displayName": "珊瑚缸",<br/>    "id": "keting-shg-64771774",<br/>    "location": "客厅",<br/>    "units": [...]<br/>  },<br/>  "msg": "获取设备组配置成功"<br/>} |



#### 3.1.3 创建设备组

| 属性         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **API接口**  | `/api/devices`                                               |
| **请求方法** | POST                                                         |
| **功能说明** | 创建新的设备组。系统自动生成唯一的设备组ID，并验证输入参数的合法性。 |
| **操作说明** | 前端填写表单后提交，传入设备组名称、显示名称、位置和单元列表（可为空数组）。服务端执行以下步骤：1) 验证必填字段；2) 将中文位置转换为拼音；3) 生成唯一ID（格式：`{位置拼音}-{设备名}-{8位随机数}`）；4) 验证单元字段和类型；5) 检查设备组名称是否重复；6) 保存到文件。 |
| **参数说明** |                                                              |

```json
{
  "name": "string",        // 设备组英文名称（必需，唯一标识）
  "displayName": "string", // 显示名称（中文，必需）
  "location": "string",    // 位置（可选，中文会自动转拼音）
  "units": "array"         // 设备单元列表（必需，可为空数组）
}
``` |
| **返回示例** | 
```json
{
  "code": 200,
  "data": {
    "name": "new_device",
    "displayName": "新设备",
    "id": "keting-new_device-12345678",
    "location": "客厅",
    "units": []
  },
  "msg": "创建设备组成功"
}
``` |
| **错误处理** | 
- 缺少必填参数：返回400错误
- 设备组已存在：返回400错误
- 单元类型无效：返回400错误 |
```

### 3.4 后端辅助功能

#### 3.4.1 请求限流

| 属性           | 说明                                                        |
| -------------- | ----------------------------------------------------------- |
| **功能说明**   | 15分钟内最多400次请求，防止恶意请求或高频调用导致服务过载。 |

## 七、架构特点

1. **轻量级设计**: 无数据库依赖，使用JSON文件存储，适合小型物联网场景
2. **纯前端实现**: 无框架依赖，部署简单，兼容性好
3. **自动ID生成**: 支持中文位置转拼音，保证ID可读性和唯一性
4. **分层架构**: 前端、API层、业务逻辑层、数据访问层分离
5. **安全性**: 包含限流、CORS、参数校验等安全措施
6. **状态管理**: 根据设备类型动态调整状态输入方式，提升用户体验

### 3.4 后端辅助功能

#### 3.4.1 请求限流

| 属性 | 说明 |
|------|------|
| **功能说明** | 防止恶意请求或高频调用导致服务过载。 |
| **配置参数** | 
- 时间窗口：15分钟（900秒）
- 最大请求数：400次
- 超过限制返回：`{"message": "请求过于频繁，请稍后再试"}` |

#### 3.4.2 CORS配置

| 属性 | 说明 |
|------|------|
| **功能说明** | 控制跨域资源共享，允许指定来源的前端访问API。 |
| **配置方式** | 通过环境变量 `CORS_ORIGINS` 设置，支持逗号分隔多个域名。未设置时允许所有来源。 |

#### 3.4.3 操作日志

| 属性 | 说明 |
|------|------|
| **功能说明** | 记录所有API操作的详细日志，包括操作时间、类型、客户端IP、操作详情和执行状态。 |
| **日志格式** | `[ISO时间戳] 操作类型 - SUCCESS/FAILED - 详情` |
| **记录内容** | 
- 操作类型：如 `GET_ALL_DEVICES`、`CREATE_DEVICE_GROUP`、`UPDATE_DEVICE_UNIT` 等
- 客户端IP：从请求头或连接信息中提取
- 操作详情：包含操作对象名称、数量、参数变更等信息 |

#### 3.4.4 参数校验

| 属性 | 说明 |
|------|------|
| **功能说明** | 确保传入的参数合法有效，防止非法数据写入。 |
| **校验规则** | 
- 必填字段检查：name、displayName、units等
- 类型校验：unit.type只能是control/state/text/data
- 数组格式校验：units必须是数组类型
- ID唯一性校验：生成ID后检查是否已存在 |

#### 3.4.5 ID唯一性保证

| 属性 | 说明 |
|------|------|
| **功能说明** | 通过循环生成和检查机制，确保生成的设备组ID和单元ID在系统中唯一。 |
| **实现方式** | 
1. 生成候选ID（包含随机数字）
2. 查询当前设备列表检查ID是否存在
3. 如果已存在，重新生成随机数
4. 重复步骤1-3直到生成唯一ID |
