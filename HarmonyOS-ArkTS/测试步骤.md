# 测试步骤

## 测试环境
- HarmonyOS 应用
- 服务器地址：192.168.1.40
- 端口：6002
- 用户名：USERNAME
- 密码：（用户提供的密码）

## 测试流程

### 1. 启动应用
- 应用启动时显示配置页面（因为 `isServerConfigured` 初始为 `false`）
- 应该看到服务器地址、端口、用户名、密码的输入框
- 应该看到登录按钮

### 2. 输入测试数据
- 服务器地址：`192.168.1.40`
- 端口：`6002`
- 用户名：xxx`
- 密码：用户密码

### 3. 点击登录按钮
预期行为：
1. 按钮调用 `saveServerConfig()` 方法
2. `saveServerConfig()` 验证输入
3. `saveServerConfig()` 调用 `ServerConfig.setServerAddress('192.168.1.40', 6002)`
4. `ServerConfig.setServerAddress()` 设置 `isConfiguredFlag = true`
5. `saveServerConfig()` 调用 `performLogin()`
6. `performLogin()` 检查 `ServerConfig.isConfigured()`（现在应该返回 `true`）
7. `performLogin()` 调用 `ApiService.login()`
8. `ApiService.login()` 构建 URL：`http://192.168.1.40:6002/api/auth`
9. 发送登录请求到服务器

### 4. 验证日志输出
预期的日志序列：
```
saveServerConfig called with serverIp: 192.168.1.40, serverPort: 6002, username: xxx
Setting server address: 192.168.1.40:6002
API Base URL set to: http://192.168.1.40:6002
ServerConfig.isConfigured() = true
Calling performLogin after successful server config
performLogin called with serverIp: 192.168.1.40, username: xxx
Checking if server is configured...
Server is configured, proceeding with login
Login URL: http://192.168.1.40:6002/api/auth
```

### 5. 可能的错误情况

#### 情况A：服务器地址为空
- 输入：空服务器地址
- 预期：显示"请输入服务端IP地址或域名"
- 日志：`Server IP is empty or contains only whitespace`

#### 情况B：无效的服务器地址格式
- 输入：`192.168.1.400`（无效IP）
- 预期：显示"服务器地址格式不正确，请输入有效的IP地址或域名"
- 日志：`Server IP format invalid: 192.168.1.400`

#### 情况C：无效端口号
- 输入：端口 `70000`
- 预期：显示"端口号必须在1-65535之间"
- 日志：`Port number invalid: 70000`

#### 情况D：空用户名或密码
- 输入：用户名或密码为空
- 预期：显示"请输入用户名和密码"
- 日志：`Username or password is empty`

#### 情况E：服务器不可达
- 输入：正确的服务器地址但服务器未运行
- 预期：显示具体的网络错误信息，而不是 `[object Object]`
- 日志：`HTTP request failed for URL: http://192.168.1.40:6002/api/auth`
- 日志：具体的网络错误信息

### 6. 成功登录后的行为
- 如果登录成功，应该跳转到主页面
- 主页面显示设备列表
- `isServerConfigured` 应该保持为 `true`
- 下次启动应用时应该直接显示主页面（如果凭据已保存）

## 调试技巧

### 查看日志
使用以下命令查看应用日志：
```
hilog -t "Index" -t "ServerConfig" -t "ApiService"
```

### 关键日志标签
- `Index`: 页面相关日志
- `ServerConfig`: 服务器配置相关日志
- `ApiService`: API 调用相关日志
- `LoginStorage`: 登录存储相关日志

### 验证服务器配置状态
检查 `ServerConfig.isConfigured()` 的返回值：
- 初始状态：`false`
- 调用 `setServerAddress()` 后：`true`

### 验证 URL 构建
检查 `ServerConfig.getApiBaseUrl()` 返回的 URL：
- 预期：`http://192.168.1.40:6002`
- 实际构建的登录 URL：`http://192.168.1.40:6002/api/auth`

## 常见问题排查

### 1. 仍然显示"服务器地址未配置"
- 检查 `saveServerConfig()` 是否被调用
- 检查 `ServerConfig.setServerAddress()` 是否成功执行
- 检查 `ServerConfig.isConfiguredFlag` 是否设置为 `true`

### 2. URL 格式错误
- 检查服务器地址是否包含空格
- 检查端口号是否为有效整数
- 检查 `ServerConfig.getApiBaseUrl()` 返回的 URL

### 3. 网络连接失败
- 确认服务器正在运行
- 确认端口 `6002` 已开放
- 确认设备可以访问服务器 IP
- 检查防火墙设置

### 4. 登录失败但服务器响应正常
- 检查用户名和密码是否正确
- 检查服务器 API 路径是否为 `/api/auth`
- 检查服务器返回的响应格式
