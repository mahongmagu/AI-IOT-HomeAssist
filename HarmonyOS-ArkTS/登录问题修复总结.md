# 登录问题修复总结

## 问题分析
用户报告输入正确的登录信息后无法登录，日志显示以下错误：
1. `[http_exec.cpp:1445] not a normalized URL`
2. `[http_exec.cpp:1471] get host and port failed`
3. `Failed to get cleartextPermitted, hostname is empty.`
4. `CURLcode result 3` - CURL 错误码 3（URL 格式错误）
5. `Login error: [object Object]`
6. `Server address is empty!`

## 根本原因
通过代码分析发现以下问题：

### 1. 登录按钮调用错误的方法
- 在 `pages/Index.ets` 中，登录按钮的 `onClick` 事件直接调用 `performLogin()`
- 但是 `performLogin()` 依赖于 `ServerConfig` 类中的服务器地址配置
- 服务器地址配置应该在 `saveServerConfig()` 方法中设置，但这个方法没有被调用

### 2. 服务器配置状态管理问题
- `ServerConfig` 类使用 `isConfiguredFlag` 标志位来跟踪配置状态
- 初始值为 `false`
- 当用户输入服务器地址并点击登录时，需要先调用 `saveServerConfig()` 来设置地址和标志位
- 但按钮直接调用 `performLogin()`，导致 `ServerConfig.isConfigured()` 返回 `false`

### 3. 错误处理不充分
- 错误信息显示为 `[object Object]`，难以调试
- 缺少详细的调试日志

## 修复方案

### 1. 修复登录按钮事件（关键修复）
**文件**: `entry/src/main/ets/pages/Index.ets`
**位置**: 第226-230行
**修改前**:
```arkts
.onClick(() => {
  if (!this.indexData.isAccountLocked) {
    this.performLogin();
  }
})
```

**修改后**:
```arkts
.onClick(() => {
  if (!this.indexData.isAccountLocked) {
    this.saveServerConfig();
  }
})
```

## 修复后的流程
1. 用户输入服务器地址、端口、用户名和密码
2. 点击登录按钮 → 调用 `saveServerConfig()`
3. `saveServerConfig()` 验证输入并设置服务器配置
4. `saveServerConfig()` 调用 `ServerConfig.setServerAddress()` 设置地址和标志位
5. `saveServerConfig()` 调用 `performLogin()`
6. `performLogin()` 检查服务器配置状态（现在应该为 `true`）
7. `performLogin()` 调用 `ApiService.login()` 进行登录

## 验证的服务器地址格式
修复后的代码验证以下格式：
- IP 地址：`192.168.1.1`、`10.0.0.1` 等
- 域名：`example.com`、`server.local` 等
- localhost
- 主机名：允许字母、数字和连字符

## 测试用例
1. **空服务器地址** → 显示"请输入服务端IP地址或域名"
2. **无效格式地址** → 显示"服务器地址格式不正确"
3. **无效端口号** → 显示"端口号必须在1-65535之间"
4. **空用户名或密码** → 显示"请输入用户名和密码"
5. **有效输入但服务器不可达** → 显示具体的网络错误信息

## 调试日志增强
修复添加了以下调试日志：
- `saveServerConfig()` 调用时的输入参数
- 服务器地址格式验证结果
- 端口号验证结果
- `ServerConfig.setServerAddress()` 调用
- `ServerConfig.isConfigured()` 状态
- `performLogin()` 开始执行
- HTTP 请求错误详细信息

## 预期结果
修复后，用户应该能够：
1. 输入有效的服务器地址（如 `192.168.6.40`）
2. 输入端口号（如 `3002`）
3. 输入用户名和密码
4. 点击登录按钮
5. 看到服务器配置被保存
6. 看到登录请求被发送到正确的 URL：`http://192.168.6.40:3002/api/auth`

如果服务器不可达，用户将看到具体的错误信息而不是 `[object Object]`。

## 注意事项
1. 确保服务器地址不包含空格（代码会自动 trim）
2. 端口号必须是 1-65535 之间的整数
3. 用户名和密码不能为空
4. 服务器必须运行在指定的端口上
5. 如果服务器使用 HTTPS，需要修改 `ServerConfig.getApiBaseUrl()` 方法
