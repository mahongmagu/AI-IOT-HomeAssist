## 四、MQTT通信协议

### 4.1 主题格式

| 主题                                                     | 方向          | 用途         |
| -------------------------------------------------------- | ------------- | ------------ |
| `iot/device/baomujian-sdj-40858013_kaiguan-2966/control` | 服务端→设备端 | 控制指令下发 |
| `iot/device/baomujian-sdj-40858013_kaiguan-2966/state`   | 设备端→服务端 | 状态上报     |

### 4.2 控制指令

| 指令  | 说明       |
| ----- | ---------- |
| `ON`  | 打开扫地机 |
| `OFF` | 关闭扫地机 |

### 4.3 状态值

| 状态  | 说明         |
| ----- | ------------ |
| `ON`  | 扫地机运行中 |
| `OFF` | 扫地机关闭   |

### 4.4 通信流程

```
服务端发布控制指令
        ↓
MQTT服务器 (192.168.1.40:1883)
        ↓
设备订阅控制主题 → 接收指令 → 执行操作
        ↓
设备发布状态 → MQTT服务器 → 服务端接收状态
```

## 五、设备控制

### 5.1 控制方式

#### 方式一：API控制

```bash
# 打开扫地机
curl -X POST http://192.168.1.40:6002/api/control-slot \
  -H "Content-Type: application/json" \
  -d '{"slot": "baomujian-sdj-40858013_kaiguan-2966", "cmd": "ON"}'

# 关闭扫地机
curl -X POST http://192.168.1.40:6002/api/control-slot \
  -H "Content-Type: application/json" \
  -d '{"slot": "baomujian-sdj-40858013_kaiguan-2966", "cmd": "OFF"}'
```

#### 方式二：MQTT直接控制

```bash
# 订阅状态主题
mqtt sub -t "iot/device/baomujian-sdj-40858013_kaiguan-2966/state" -h 192.168.1.40 -p 1883 -u xx -P *******

# 发布控制指令
mqtt pub -t "iot/device/baomujian-sdj-40858013_kaiguan-2966/control" -m "ON" -h 192.168.1.40 -p 1883 -u xx -P *******
```

#### 方式三：Web界面控制

1. 打开设备控制页面：`http://192.168.1.40:6002/index.html`
2. 点击设备组"扫地机"
3. 点击"打开"或"关闭"按钮

## 七、OTA固件更新

### 7.1 OTA配置

设备支持Arduino OTA远程固件更新：

```cpp
// OTA配置
ArduinoOTA.onStart([]() {
  Serial.println("OTA更新开始");
  digitalWrite(RELAY_PIN, LOW);  // 更新前关闭继电器
});

ArduinoOTA.onEnd([]() {
  Serial.println("\nOTA更新完成");
});

ArduinoOTA.onProgress([](unsigned int progress, unsigned int total) {
  Serial.printf("OTA进度: %u%%\r", (progress / (total / 100)));
});

ArduinoOTA.begin();
```

### 7.2 更新方法

**方法一：使用Arduino IDE**

1. 确保设备和电脑在同一网络
2. 在Arduino IDE中选择"Tools" -> "Port" -> 选择OTA端口（如 `esp8266-xxxx at 192.168.1.xxx`）
3. 点击"上传"按钮

**方法二：使用命令行**

```bash
arduino-cli upload -p esp8266-ota://192.168.1.xxx --fqbn esp8266:esp8266:generic ESP8266_IoT.ino
```

### 7.3 OTA错误码

| 错误码 | 说明                         |
| ------ | ---------------------------- |
| 0      | OTA_AUTH_ERROR - 认证失败    |
| 1      | OTA_BEGIN_ERROR - 开始失败   |
| 2      | OTA_CONNECT_ERROR - 连接失败 |
| 3      | OTA_RECEIVE_ERROR - 接收失败 |
| 4      | OTA_END_ERROR - 结束失败     |

## 八、故障排除

### 8.1 常见问题

| 问题             | 原因                     | 解决方案               |
| ---------------- | ------------------------ | ---------------------- |
| 设备无法连接WiFi | WiFi密码错误或信号弱     | 检查WiFi配置和信号强度 |
| MQTT连接失败     | MQTT服务器地址或端口错误 | 确认MQTT服务器配置     |
| 控制指令无响应   | 设备未订阅控制主题       | 检查代码中的订阅逻辑   |
| 状态不更新       | WebSocket连接断开        | 检查状态服务是否运行   |
| OTA更新失败      | 网络不稳定               | 确保网络稳定，重试更新 |
